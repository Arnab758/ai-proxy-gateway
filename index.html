FAQ

Frequently asked questions

How does semantic caching work? ▼
Proxymic converts each request into an embedding vector and compares it against previously cached responses. When a new request is semantically similar (above 95% threshold), we serve the cached response instead of calling the LLM API. This catches paraphrased questions, similar code patterns, and repeated queries — all without your users noticing any difference.
Will this affect the quality of responses? ▼
No. When we serve a cached response, it's an identical response to what the LLM would have returned. The similarity threshold is configurable — you can set it as high as 99.9% for strict matching, or lower for more aggressive caching. During the 4-day trial, we only observe your traffic and show what would have been cached, so you can verify the quality before committing.
What LLM providers do you support? ▼
Any OpenAI-compatible API. This includes OpenAI, Groq, Anthropic via their API-compatible endpoints, Together AI, Fireworks, Perplexity, and more. We support primary + fallback providers with automatic failover and circuit breaker.
What's the difference between observer mode and active caching? ▼
In observer mode (free trial), Proxymic watches your traffic silently. It records which requests would have been cache hits, how many tokens you would have saved, and your projected cost savings — but it always forwards requests to the LLM. In active mode (Pro plan), we actually serve cached responses, cutting your API calls by 40-70% immediately.
How do I deploy Proxymic? ▼
The easiest way is one-click deploy on Railway. You can also run it locally with Docker. Full deployment guide is on our GitHub README.
What happens when my trial ends? ▼
Your trial dashboard shows all the savings data collected during the 4-day period. Your app continues to work normally — the proxy simply stops tracking metrics. To continue saving, upgrade to a Pro plan. We'll send a reminder email before your trial expires.
